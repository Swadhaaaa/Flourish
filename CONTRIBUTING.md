# Contributing to Flourish

Thank you for your interest in contributing to Flourish! We welcome contributions from the community to help make Flourish a more robust, professional, and accessible platform.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please report any unacceptable behavior to the project maintainers.

## How to Contribute

### 1. Reporting Bugs
- Search existing issues to ensure the bug hasn't already been reported.
- Open a new issue with a clear title, description, steps to reproduce, and screenshots if applicable.

### 2. Suggesting Enhancements
- Open an issue explaining the proposed feature, its benefits, and how it aligns with the project goals.

### 3. Submitting Pull Requests
- Fork the repository and create your branch from `main`.
- Write clear, documented code.
- Ensure all tests pass and there are no linting or formatting errors.
- Commit messages should be clear and follow standard conventions (e.g., `feat: add Google Calendar sync`, `fix: correct token renewal expiry`).
- Submit a Pull Request (PR) describing the changes and referencing any related issues.

## Local Development Setup

Refer to the [Developer Guide](docs/DeveloperGuide.md) for detailed instructions on configuring your local workspace for frontend and backend development.

## Style Guidelines

- **Frontend**: Follow standard React & TypeScript best practices. Code formatting is enforced via Prettier and ESLint.
- **Backend**: Adhere to PEP 8 standards for Python. Use type hints where appropriate.
